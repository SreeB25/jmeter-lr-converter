#  JMeter → LoadRunner (VuGen) Script Converter Plugin

A JMeter GUI plugin to convert `.jmx` test plans into **LoadRunner Web HTTP/HTML scripts** with parameter files and basic correlations.

**Status:** Development

## Requirements

- Java 21 (LTS) — project has been updated to target Java 21
- Maven 3.6+

## Build

Set `JAVA_HOME` to a Java 21 JDK and build with Maven:

```powershell
$env:JAVA_HOME = 'C:\Path\To\jdk-21'    # adjust for your system
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
mvn clean package
```

To skip tests during a quick build:

```powershell
mvn -DskipTests package
```


---

##  Features

- Converts **Thread Groups** to LoadRunner script folders
- Converts JMeter variables `${var}` → `{var}` (LoadRunner syntax)
- Copies CSV test data and generates `.dat` files automatically
- Converts:
  - `RegexExtractor` → `web_reg_save_param_ex`
  - `JSONPostProcessor` → `web_reg_save_param_json`
- Shows progress logs in JMeter UI
- Plugin signature footer: **SreeBommakanti**

## Notes about the Java 21 upgrade

- The `pom.xml` was updated to target Java 21 using the Maven Compiler Plugin with `<release>21</release>` so compilation and generated bytecode target Java 21.
- Ensure your local environment runs JDK 21 when building or running the plugin.

## Running (for development)

- Install JMeter matching the target runtime for plugin development (the `pom.xml` references JMeter `5.6.3`).
- Place the built jar into JMeter's `lib/ext/` directory or use the JMeter plugin development workflow.
